TabooTyper Package Documentation

Welcome to the documentation of the TabooTyper package. Below you will find the instructions on how to use the package along with the explanations of its settings.

Installation

To get started with the TabooTyper package, you need to include the JavaScript file in your project. The simplest way to do so is by including the CDN link directly in your HTML.

<script src="https://cdn.jsdelivr.net/gh/sumeetghimire/TabooTyper/index.js"></script>

Place this script tag just before the closing </body> tag in your HTML file to ensure it loads after the page content.

How to Use

To use the TabooTyper package, you need to include the JavaScript file in your project and initialize the TabooTyper class with the appropriate settings.

Settings Explanation

The TabooTyper class accepts the following settings:

1. selector

The CSS selector that targets the elements where TabooTyper will be applied (e.g., textarea, input).

selector: "textarea, input"

2. bannedWords

An object where keys are the words you want to ban and values are their replacements. Example:

bannedWords: {
            "dumb": "not smart",
            "stupid": "silly",
            "hate": "dislike",
            "bad": "unpleasant"
        }

3. masked

Set to true to replace the banned words with a masking character. Set to false to allow replacement without masking.

masked: false

4. maskCharacter

The character to use for masking the banned words. Default is "*".

maskCharacter: "#"

5. caseSensitive

If set to true, the check for banned words will be case-sensitive. Set to false for a case-insensitive check.

caseSensitive: false

6. trackFrequency

Set to true to track how often each banned word appears. This can be useful for analytics.

trackFrequency: true

7. highlight

If set to true, banned words will be highlighted in the text. Example:

highlight: true

8. suggestions

If set to true, users will be shown alternative words to replace the banned word with.

suggestions: true

9. contextSensitive

Set to true to enable context-sensitive replacements. This means the system will consider the surrounding text before replacing a word.

contextSensitive: true

10. warningStyle

Set a custom style for the warning popup that appears when a banned word is detected. Example:

warningStyle: "color: red; font-weight: bold;"

11. popupBackgroundColor

The background color of the warning popup. Default is red. Example:

popupBackgroundColor: 'blue'

12. popupTextColor

The text color of the warning popup. Default is white. Example:

popupTextColor: 'yellow'
